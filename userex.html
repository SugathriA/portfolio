Voice user interface for data center operations

What is the significance of this Project?

Cisco Virtual Topology System VTS, is a product that automates data center operations. In the existing UI , user has to traverse the navigation to configure the data center overlay objects and deal with complex UI forms. There is a huge scope to improve the user experience by using natural langugage assistant in this product.

Who are the target audience and what is the design problem being solved?

In the current VTS UI, Service Provider Administrator (SP Admin such as Verizon, Telstra) is the target user.

The problem that is being solved is to make any overlay object configuration simple. Using the current VTS UI, it is quite a long process. To better explain this I am using, an example of the task of attaching port to network. Attaching port to a network enables the network (overlay object) in a data center.

To achieve this user has to:

  • Select the tenant.
  • Select the desired network.
  • Edit the network.
  • Initiate port attach.
  • Select the device.
  • Select the port on device.
  • Select the baremetal to which it has to be attached to.
  • Select the port on baremetal.

This involves traversal, clicks, selections to find the right port. Below are some screenshots from UI depicting the flow. While doing these steps if any of the devices are down, user cannot be notified right away.

Network port attach flow

What is the proposed solution and what is my role in it?

The proposed solution was designing a mobile application that is voice enabled. This natural language voice assistant app lets the user accomplish this task without having to navigate and find the right network and port. All user needs to do is take the assistance of app prompts and respond.

I am the lead designer for VTS. It was originally my idea to innovate the current product with natural voice assistance. I supported my idea with this video as a proof of concept.

What is cool about this solution?

This solution helps in overcoming the complexity of managing data center objects.

  • Interactive natural language voice assistant.
  • Prompts for information to supply or initiate the next command.
  • Displays context of the command and status of the object.
  • User can view the interactions of each step in a visual way and intuitively complete it.
  • Eliminates complex traversal in the navigation to get to the right object to configure.
  • Eliminates the need to go to CLI or use complex UI forms.
  • Provides ability to see information in-context, and know if the task can be successful much earlier.
  • Cisco patent was filed for this design.

User interface matching real world for data center operations

What is the significance of this Project?

Cisco Virtual Topology System (VTS), is a product that automates data center operations. There was a feature request that allows users to group multiple interfaces across data center devices and assign ranges. This allows the user in spawing Virual Machines later in those ranges. This is a very data intense, complex problem to solve and the solution has to scalable irrespective of the number of devices in a data center.

Who are the target audience and what is the design problem being solved?

In the current VTS UI, Service Provider Administrator (SP Admin such as Verizon, Telstra) is the target user.

The problem that is being solved is to create interface group in a simple way. Below are some numbers that explain the challenge of this problem which is scale

  • A data center can have 156 devices (switches).
  • Each switch can have 1-13 modules.
  • Each module has 48 ports also called as interfaces.

User can select any number of devices and any number of ports and define any number of ranges. Also in a scenario where user needs to edit, information related to devices, interfaces and ranges should be displayed in order to make any changes.

What is the proposed solution and what is my role in it?

The proposed solution was to build a work flow in the UI. Workflow addressing the problem and visual represention of the interfaces in both define and edit scenarios. The steps user needs to take in the workflow are:

  • Define a group.
  • Select one or more devices.
  • Select one or more interfaces per device.
  • Assign some ranges.

My role in providing this solution is:

  • White boarding to gather these requirements.
  • Understand the scenarios.
  • Story boarding the flow.
  • Providing several design options.
  • Discussing options with product team, architects and developers.
  • Iterating designs based on their feedback.
  • Providing high fidelity Mock ups.
  • Redline specs for the final design.

What is cool about this solution?

This work flow solution helps in dealing with the scale as it is broken in to several steps. At each step user is only dealing with one step of data. Also the visual representation of selected device ports, helps user identify them quickly. It also reduces the clutter on UI. Cisco patent was filed for this design.

Information Architecture for strong navigation system

What is the significance of this Project?

Cisco Virtual Topology System (VTS), is a product that automates data center operations. Many new features are added to the product at a fast pace. Everytime new features are added user being able to find them with ease is critical in the product. This was a task of validating the current navigation with the users and propose any changes where they are necessary.

Who are the target audience and what is the design problem being solved?

In the current VTS UI, Service Provider administrator (SP Admin such as Verizon, Telstra) is the target user.

The problem that is being solved is making navigation disoverable, accessible and contextual. New and returning users should be able to navigate through product with ease.

What is the proposed solution and what is my role in it?

The proposed solution was to put forward a navigation that is consistent, strives for flat navigation and imitates the structure of data center in real world.

My role in this exercise was :

  • Conducting in-person interviews.
  • Using online Optimal cardsort technique and anyzing the results provided by all users.
  • Based on the results proposing an efficient navigation system.

What is cool about this solution?

This navigation enhaces the usability of the product. This makes the product very simple, usable and lets user accomplish tasks efficiently. Lot of back and forth is avoided in the provided solution and all the UI problems buried underneath because of platform constraints are addressed.
