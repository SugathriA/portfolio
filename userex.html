What is the significance of this Project?

Cisco Virtual Topology System (VTS), is a product that automates data center operations. In the existing UI for Cisco VTS, user has to traverse the navigation to get the overlay object that needs to be managed and deal with complex UI forms. There is a huge scope to improve the user experience by using natural langugage assistant in this product.

Who are the target audience and what is the design problem being solved?

In the current VTS UI, Service Provider administrator (SP Admin such as Verizon, Telstra) is the target user.

Any overlay object configuration is complicated to solve. As an example I am using the task of attaching port to network. Attaching port to a network enables the network in a data center.
To achieve this user has to:

  • Select the tenant.
  • Select the desired network.
  • Select the device.
  • Select the port on device.
    •  
This involves many traversal, clicks, selections to find the right port.Below are some screenshots from UI depicting the flow.

What is the proposed solution and what is my role in it?

The proposed solution was designing a mobile application that is voice enabled. This natural language voice assistant app lets the user accomplish this task without having to navigate and find the right network and port. All user needs to do is take the assistance of app prompts and respond.

I proposed this idea and came up with this video as a proof of concept.

What is cool about this solution?

This solution helps in overcoming the complexity of managing data center objects with an interactive natural language voice assistant where users can initiate a task. The applicaion then prompts for information to supply or initiate the next command. It displays context of the command and status of the object. That way user can view the interactions of each step in a visual way and intuitively complete it. This eliminates complex traversal in the navigation to get to the right object to configure, eliminates the need to go to CLI or use complex UI forms, gain an ability to see information in-context, and know if the task can be successful much earlier.Cisco patent was filed for this design.

What is the significance of this Project?

Cisco Virtual Topology System (VTS), is a product that automates data center operations. There was a feature request that allows users to spawn Virtual Machines(VMs). This is a very data intense, complex problem to solve and the solution has to scalable irrespective of the number of devices in a data center.

Who are the target audience and what is the design problem being solved?

In the current VTS UI, Service Provider administrator (SP Admin such as Verizon, Telstra) is the target user.

A data center typically consists of devices (switches). Each switch can have 1-13 modules. Each module has 48 ports also called as interfaces. The challange in this problem is the scale. There could be up to 156 devices in a data center. User can select any number of devices and any number of ports and define any number of ranges. Also in a scenario where user needs to edit, information related to devices, interfaces and ranges should be displayed in order to make any changes.

What is the proposed solution and what is my role in it?

The proposed solution was to build a work flow in the UI that addresses the problem and visual represention of the interfaces in both Define and Edit scenarios. User has to first define a group. Select one or more devices. Select interfaces per device. Assign some ranges. Now VMs can be spawned in this range in the data center.

My role in providing this solution is white boarding to gather these requirements and understand the scenarios. Story boarding the flow. Providing several design options and discussing with product team, architects, developers and iterating designs based on their feedback. Providing high fidelity Mock ups and redline specs for the final design.

What is cool about this solution?

This work flow solution helps in dealing with the scale as it is broken in to several steps. At each step user is only dealing with one step of data. Also the visual representation of selected device ports, helps user identify them quickly. Cisco patent was filed for this design.

What is the significance of this Project?

Cisco All In One (AIO) is a product that is used to manage Cisco Virtually Managed Services (VMS) deployment environments. The current UI had to be redesigned enhancing the user experience.

Who are the target audience and what is the design problem being solved?

In the current AIO UI, VMS deployments administrator is the target user.

There are many usability issues, pain points in the current UI. Many tasks are not in context. No consistency in language. No help on the complex UI forms. Lot of data on one page with no search functionality. All these issues have to be addressed and an efficient and enjoyable experience should be built.

What is the proposed solution and what is my role in it?

The proposed solution was to show an overview of AIOs that helps user to make decisions on what AIO should be used. Moving all actions related to AIO at one place. Cleaning up the forms and adding ghost text. Combining the related tasks to simplify and avoid clutter in design.

My role in providing this solution is identifying pain points using heuristic evaulation. Conducting task analysis to realize efficiency of each task. Coming up with several design options and conducting usability studies with the users. Iterating to provide final design.

What is cool about this solution?

The solution is not only easy to use but has greatly benefited the admin users in quickly identifying the isssues with AIOs.

In the current VTS UI, Service Provider administrator (SP Admin such as Verizon, Telstra) is the target user.

What is the proposed solution and what is my role in it?

The proposed solution was to put forward a navigation that is consistent, strives for flat navigation and imitates the structure of data center in real world.

My role in this exercise was conducting in-person interviews, using online Optimal cardsort technique and anyzing the results provided by all users. Based on the results proposing an efficient navigation system.

What is cool about this solution?

This navigation enhaces the usability of the product. This makes the product very simple, usable and lets user accomplish tasks efficiently. Lot of back and forth is avoided in the solution and all the UI problems buried underneath because of platform constraints are addressed.
